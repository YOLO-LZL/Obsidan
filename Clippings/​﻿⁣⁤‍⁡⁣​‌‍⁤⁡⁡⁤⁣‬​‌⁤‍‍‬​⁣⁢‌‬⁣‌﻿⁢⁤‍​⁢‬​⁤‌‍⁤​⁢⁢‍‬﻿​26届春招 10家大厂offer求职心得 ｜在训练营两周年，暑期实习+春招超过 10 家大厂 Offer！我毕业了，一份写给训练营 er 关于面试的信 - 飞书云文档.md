## 四、暑期实习 vs 春招，到底有什么不一样

我去年暑期实习面了 45 场，今年春招面了 40 多场。两轮下来，体感很不一样：

**第一，暑期实习更看基础，春招更看项目深度。**&#x6691;期实习的面试，八股文占比明显更高。很多一面甚至二面，大部分时间都在问 Java 并发、MySQL 索引、Redis 缓存这些基础知识。面试官的态度是：你是实习生，我不指望你做过多复杂的系统，但基础得扎实。春招就不一样了。面试官默认你有实习经历，会直接追着项目细节问：你为什么做这个选择、遇到问题怎么解决的、如果重新设计你会怎么改。八股还是会问，但比例明显下降。

**第二，春招的 HR 面会深入很多。**&#x6691;期实习的 HR 面比较轻松，主要确认实习时间、工作意向。春招的 HR 面会深入很多——问职业规划、问你怎么看加班文化、问你在多个 offer 之间怎么选，甚至会问家庭情况。

**第三，暑期实习战线更长，但四月是黄金窗口。**&#x6691;期实习从二月底到五月初都有机会。大厂的暑期实习岗位通常四月中下旬开始收口。如果你是 27 届的同学，**四月是你的黄金时间**。

**第四，AI/Agent 方向是今年春招的新变量。**&#x53BB;年暑期实习几乎没人问 AI 相关的内容。今年春招，至少一半的面试会问「你用 AI 写代码吗」「你怎么看 AI Coding」。如果你投 Agent 岗位，RAG 流程、Agent 设计、评测体系是必考题。这块我会后面详细讲解

## 五、后端面试高频题汇总

以下所有问题来自我近 90 场面试的真实记录。括号里是被问到的总次数，⭐ 是重要性评级。

### Java 并发编程（几乎每场都问）

1. **线程池核心参数和原理**（17 次）⭐⭐⭐⭐⭐核心线程数、最大线程数、等待队列、拒绝策略。追问：IO 密集型和 CPU 密集型怎么配？无界队列的坑？

2. **synchronized 和 ReentrantLock**（7 次）⭐⭐⭐⭐三种加锁方式、锁升级、和 ReentrantLock 的区别。

3. **CAS 与 ConcurrentHashMap**（8 次）⭐⭐⭐⭐CAS 原理、ABA 问题、ConcurrentHashMap 为什么同时用 CAS 和 synchronized。

4. **ThreadLocal 原理和内存泄漏**（6 次）⭐⭐⭐⭐为什么 key 用弱引用 value 用强引用、内存泄漏怎么规避。

5. **volatile 关键字**（5 次）⭐⭐⭐⭐可见性和有序性、为什么 DCL 必须用 volatile。

6. **CompletableFuture**（2 次）⭐⭐⭐⭐怎么用、有没有指定线程池。

## 六、Agent / AI 方向面试高频题

如果你投 Agent 开发或大模型应用岗，这部分是必考区域。

### 我是怎么学习 AI/Agent 的

先说结论：**AI 方向不能只看八股，必须动手做项目。**&#x6211;的学习路径分三步：

**第一步：建立概念框架**

* 把 RAG 的完整流程搞清楚：文档加载 → 分块 → 向量化 → 存储 → 检索 → 生成

* 理解 Agent 的核心循环：感知 → 规划 → 行动 → 反馈

* 看了几篇经典论文（ReAct、Self-RAG），不用精读，了解思想就行

**第二步：做一个完整项目**

* 我做的是 RAG 问答智能体，从最简单的版本迭代到生产可用级别

* 踩过的坑才是面试素材：为什么 naive RAG 效果差？chunk size 怎么选？向量召回和关键词召回各有什么问题？混合召回怎么融合排序？

* **建议**：不要用现成框架一键搭建就完事，至少核心流程自己实现一遍，面试追问才不会露馅

**第三步：关注前沿动态（持续）**

* 关注 AI Coding 工具的发展（Cursor、Claude Code、GitHub Copilot），自己日常开发中用起来

* 关注 Agent 框架的演进（LangChain、CrewAI、AutoGen），了解设计理念的差异

* 看看大厂的技术博客，了解工业界在怎么落地 RAG 和 Agent

## 七、手撕代码题汇总

90 场面试中大概有 30 场要现场写代码。

### 我的算法学习心得

算法是很多人最头疼的部分，我也不例外。分享一下我的方法：

**第一阶段：建立题感**

* 先刷 LeetCode hot100，不要求自己每题都独立做出来

* 看不懂的题先看题解，理解思路后自己默写一遍

* 这个阶段的目标是建立「看到题目能归类到哪种算法」的直觉

**第二阶段：反复巩固**

* hot100 刷第二遍，这一遍要求独立完成

* 做不出来的题标记下来，第三遍只刷标记的题

* 同时开始刷 CodeTop，按照目标公司的高频题刷

**第三阶段：保持手感**

* 面试期间每天 1-2 道，不贪多，重在保持状态

* 面试前一天不刷新题，只复习做过的高频题

* **重点练手写**：面试手撕是白板或在线编辑器，和本地 IDE 体验完全不同，要习惯没有自动补全的环境**核心原则**：链表和滑动窗口这两类题投入产出比最高，面试出现频率高且解法相对固定。DP 题除了股票系列和背包，其他的面试中出现概率不大，不用花太多时间。

## 八、面试心得

### 简历不是履历，是提问清单

你写上去的每一行，都要做好被追问三层的准备。写不清楚、答不上来的东西，不如不写。

* **项目描述必须有量化数据。** 不是「优化了性能」，是「QPS 从 X 提升到 Y」。

* **技术栈写上去的每一个都要能答。** 写了 Redis 就得答分布式锁和缓存一致性，写了线程池就得准备核心参数和拒绝策略。

* **实习经历是主战场。** 80% 的技术面都围绕实习项目深挖。

### 不懂装懂是面试最致命的减分项

有一次 HR 面反馈说我「底层原理不扎实，有不懂装懂的嫌疑」。面试官能非常敏锐地感知到你是真懂还是在编。坦诚说「这块我不太熟，但我理解是……」反而比硬编一个答案强得多。

### 面试官追问不是在为难你

他是在测试你的能力边界。「为什么选这个方案而不是那个」比「你用了什么方案」重要十倍。

### 关于表达能力

技术决定你能不能通过，**表达决定你拿什么级别的评价**。我前期面试最大的问题是想到什么说什么，回答发散没有结构。后期刻意训练结构化表达后，通过率明显涨了。**我具体是怎么刻意练习的：**&#x6211;的方法比较"技术流"——用 **LazyType（语音转文字工具）** 结合 AI 做口述训练：https://lazytyper.com/

1. **口述录音**：对着面经里的问题，用嘴巴把答案讲一遍，LazyType 实时转成文字

2. **对比精修稿**：把口述的文字和我之前用 Claude Code 精修过的标准答案放在一起对比。哪里逻辑跳跃了、哪里关键信息没说到、哪里废话太多，一目了然

3. **反复迭代**：针对差距大的问题和 CC 等智能体讨论，指导 AI 觉得 ok 为止这个方法的好处是**可量化**——不是凭感觉觉得自己"好像讲得还行"，而是白纸黑字地看到口述质量和标准答案之间的差距。练了一两周之后，面试时的表达流畅度和结构感有明显提升。

### 谈薪经验

* **只报期望上限，不给区间。** 报了「XX 到 YY」，HR 立刻锁定下限。

* **不直接说 OK。** 即使符合预期也说「我需要考虑一下」。

* **聚焦 base。** 签字费是一锤子买卖，base 才是未来涨薪的基数。

* **真诚才是必杀技。** 谈薪之前先让 HR 觉得你真的可能签他们。说一下这个工作哪里吸引你，有动机的真诚比嘴上说「我很想去」管用得多。

### 写简历的心得

简历不是流水账，是面试的提问清单。你写上去的每一行，都要做好被追问三层的准备。

#### 简历结构（四大件）

一份后端简历核心就四个模块，按重要性排序：

1. **教育背景**

2. **实习经历**

3. **项目经历**

4. **技术栈**

背景简单干净就好，**不需要太花里胡哨的模板**。花哨的排版反而显得不专业，面试官看简历就看内容，模板越简洁越好。

#### 个人信息 & 头像

* **必填信息**：姓名、电话、邮箱、个人博客（有技术博客是加分项）、籍贯

* 籍贯别觉得没用——万一面试官是老乡，距离感一下就拉近了，聊起来会轻松很多

* **贴上头像**，一般放右上角。有照片的简历辨识度更高，HR 筛选的时候印象更深

#### 实习 & 项目的写法

实习经历和项目经历都分成两部分写：**第一部分：简介**（1-2 句话）

* 用最通俗的语言说清楚这个项目/实习是做什么的

* 比如：「xx汽车数据平台组，负责内部数据看板的后端开发」**第二部分：职责描述**（分点展开，3-5 条）

* 每一条用「动词开头 + 具体做了什么 + 量化结果」的格式

* 比如：「设计并实现基于 Redis 的分布式锁方案，解决库存超卖问题，QPS 从 800 提升到 3200」

* 每一条都是一个可以被追问的"钩子"

#### 几个核心原则

1. **量化一切**：不是「优化了性能」，是「QPS 从 800 提升到 3200，p99 延迟从 200ms 降到 45ms」。没有数字的简历就是没有说服力的简历

2. **突出"我做了什么"而非"项目是什么"**：面试官想看的是你的贡献，不是项目介绍。用「负责/设计/实现/优化」开头，不要用「该项目是一个…」

3. **技术栈要诚实**：写了的就必须能答上来。写了 Redis 就得准备分布式锁和缓存一致性，写了线程池就得准备核心参数和拒绝策略。宁可少写，不要写了答不上来

4. **项目按 STAR 法则组织**：Situation（背景）→ Task（任务）→ Action（你做了什么）→ Result（成果和数据）

5. **留白引导追问**：在简历中适当留一些"钩子"，让面试官顺着你的节奏问。比如写「使用混合召回策略提升检索准确率」，面试官大概率会问「什么混合召回」——这就进入了你准备好的领域

### 八股一定要系统过一遍

Java 并发、MySQL、Redis 三块加起来覆盖 80% 以上的面试。不需要每个知识点都能写论文，但线程池参数、索引原理、分布式锁这些必须能用自己的话讲清楚。训练营的题库背完已经战无不胜，推荐大家系统学习：[训练营八股题库](https://ls8sck0zrg.feishu.cn/wiki/wikcn70G9T95SVI0B0vbYpqns7f) 背完之后有空余时间就广泛阅读**课余推荐阅读：**

* **《Effective Java》**：Java 进阶必读，很多最佳实践面试会直接问到

* **《Clean Code》**：代码规范入门，养成好习惯，Code Review 的时候也用得上

* **《黑客与画家》**：拓宽视野，理解程序员这个职业的底层逻辑，面试聊到职业规划时也能有独到的输出

### 手撕题每天保持手感

不用刷很多，每天 1-2 道，重点把链表、滑动窗口、动态规划、回溯刷熟。面试的时候能在 15 分钟内写出来就够了。hot100 刷三遍绝对够用。

### 关于 AI 方向的判断

一位前辈跟我说过：Agent 到现在还没带来颠覆性营收，风口退了这个岗位可能会尴尬。最稳的简历组合是 **大厂实习 + AI 项目经验 + 扎实的业务系统能力**。AI 是加分项，但不能是唯一的标签。

### 三年视角做决策

第一年薪资差几万意义不大。该算的是三年后你值多少钱、能去哪里。一位学长的话到现在都记得：「你说的那些 AI 工具，coding assistant、RAG、Agent，一年后可能有另一个覆盖面更大的。重要的是工具还是它们要做的事？」**AI 工具是提效手段，但团队活下去靠的是业务价值。**

### 应投尽投，越早越好

投的晚可能面试机会都没有。我同学暑期实习等到 5 月才投美团，直接三个志愿简历全挂。大胆投大厂——个人经验，顶级大厂反而相对愿意培养人，中厂更喜欢拿来即用。

### 先快速通学，再反复整理

第一遍是最痛苦的，很多知识点非常难啃。但坚持过后，第二遍只需要第一遍 10-30% 的时间。到第三遍基本融会贯通。优先完成八股核心知识的第一遍（JVM、集合、并发、MySQL、Redis），心里先有脉络，算法配合着慢慢来，每天一两道。快面试的时候再大量刷题加深印象。

### 口述复盘是王道

一定要用口述的方式把所有面试题讲一遍。口述什么水平，面试临场发挥基本就是什么水平。如果口述没问题，这个知识点就不会出大问题。
